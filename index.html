OpenClaw en Windows 11

Instalación nativa con PowerShell — Sin Linux, sin WSL2

⊞ Windows 11 Nativo
PowerShell 7+
RTX 4060 · 8GB VRAM · 32GB RAM
NemoClaw: solo parcial en Windows
⚠️
Limitación importante de Windows nativo: NemoClaw (el sandbox de seguridad de NVIDIA) actualmente no tiene soporte completo en Windows. Funciona solo en Linux nativo o WSL2. En esta guía usaremos Docker Desktop como capa de aislamiento alternativa, que ofrece seguridad similar pero sin el runtime OpenShell de NVIDIA. Si quieres NemoClaw completo, usa la guía WSL2.

Antes de instalar cualquier cosa

winver

# O via PowerShell

Verificar que los drivers NVIDIA estén instalados

# Verificar GPU y drivers NVIDIA

# Si nvidia-smi no existe, descargar drivers desde:
# https://www.nvidia.com/Download/index.aspx
# Buscar: RTX 4060 → Windows 11 → Game Ready Driver
ℹ️
Habilitar virtualización (requerida para Docker)

Abre el Administrador de tareas → Rendimiento → CPU y verifica que dice "Virtualización: Habilitada". Si no, debes habilitarla en la BIOS/UEFI (busca "SVM Mode" o "Intel VT-x").

1
Instalar PowerShell 7
La versión moderna de PowerShell (no la que viene por defecto en Windows)
ℹ️
¿Por qué PowerShell 7 y no el default?

Windows incluye PowerShell 5.1 por defecto, que es antigua. PowerShell 7 (basada en .NET 8) es más rápida, tiene mejor soporte para JSON y comandos modernos que OpenClaw usa.

En PowerShell 5.1 (la que ya tienes, ejecutar como Administrador)
# Instalar PowerShell 7 via winget
winget install --id Microsoft.PowerShell --source winget

# Después de instalar, abrir PowerShell 7 (aparece como "PowerShell 7" en el menú inicio)
# Verificar versión
$PSVersionTable.PSVersion
⚠️
Todos los comandos siguientes son para PowerShell 7

Abre PowerShell 7 (no Windows PowerShell 5.1) para todos los pasos siguientes. Puedes buscarlo en el menú inicio o fijarlo en la barra de tareas.

2
Instalar herramientas con winget
El gestor de paquetes oficial de Windows
PowerShell 7 — Ejecutar como Administrador
# Actualizar winget a la última versión
winget upgrade --all

# Instalar Git (necesario para npm y skills)
winget install --id Git.Git -e --source winget

# Instalar Windows Terminal (mejor experiencia de terminal)
winget install --id Microsoft.WindowsTerminal -e

Instalar Node.js 24
Runtime requerido por OpenClaw
Opción A — winget (recomendada)
# Instalar Node.js 24 LTS
winget install --id OpenJS.NodeJS.LTS -e

# Cerrar y reabrir PowerShell 7, luego verificar

Opción B — Instalador gráfico

Ve a nodejs.org → descarga el instalador .msi de Node.js 24 LTS → ejecuta y sigue el asistente.

⚠️
Política de ejecución de scripts

Si npm da error de "scripts deshabilitados", ejecuta esto en PowerShell 7 como Administrador:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Instalar Docker Desktop
Reemplaza a NemoClaw en Windows — proporciona aislamiento de contenedores
PowerShell 7 como Administrador
# Instalar Docker Desktop via winget
winget install --id Docker.DockerDesktop -e

Después de instalar, abre Docker Desktop y en Settings:

  • General → Activa "Use the WSL 2 based engine" (más rápido)
  • Resources → Advanced → Asigna al menos 8 GB RAM al motor Docker
  • Espera a que el ícono de Docker en la bandeja del sistema sea verde
Verificar Docker funcionando
# Verificar instalación
docker --version
docker run hello-world
5
Instalar Ollama con soporte GPU
Runtime para modelos locales — detecta la RTX 4060 automáticamente
Opción A — winget
# Instalar Ollama
winget install --id Ollama.Ollama -e
Opción B — Instalador gráfico

Ve a ollama.com/download → descarga OllamaSetup.exe → ejecuta el instalador.

Ollama en Windows corre como servicio en background automáticamente. Verás el ícono de la llama en la bandeja del sistema.

Verificar GPU activa
# Verificar que Ollama está corriendo
Invoke-RestMethod -Uri "http://localhost:11434/api/tags"

# Verificar que usa la GPU (aparece NVIDIA en el output)
nvidia-smi

# Variables de entorno para VRAM (opcional pero recomendado)
# Agregar a variables de entorno del sistema:
# OLLAMA_GPU_OVERHEAD = 512MB  (reserva para el OS)
# OLLAMA_MAX_VRAM = 7680MB     (deja 512MB libre de los 8GB)

Configurar variables de entorno de Ollama (permanente)

# Configurar variables de entorno del sistema (como Administrador)
[System.Environment]::SetEnvironmentVariable("OLLAMA_GPU_OVERHEAD", "512MB", "Machine")
[System.Environment]::SetEnvironmentVariable("OLLAMA_MAX_VRAM", "7680MB", "Machine")

# Reiniciar el servicio Ollama para aplicar
Restart-Service -Name "ollama" -ErrorAction SilentlyContinue
# Si no hay servicio, cierra y reabre Ollama desde el menú inicio

Ubicación de archivos en Windows

En Windows, el directorio de configuración de OpenClaw es C:\Users\TuUsuario\.openclaw\ en lugar de ~/.openclaw/ de Linux. En PowerShell puedes acceder con $env:USERPROFILE\.openclaw\

Descargar modelos locales
Optimizados para RTX 4060 8GB VRAM + 32 GB RAM
PowerShell 7
# Modelo principal — 100% en GPU, mejor tool calling
# VRAM: ~5.5 GB | Velocidad: ~65 tok/s | Contexto: 128K
ollama pull qwen3:8b-q4_K_M

# Modelo de código — fine-tuned para programación
# VRAM: ~4.5 GB | Velocidad: ~75 tok/s | Contexto: 128K
ollama pull qwen2.5-coder:7b-q4_K_M

# Modelo heavy — para razonamiento complejo (offloading a RAM)
# VRAM: ~8+2 GB | Velocidad: ~40 tok/s
ollama pull qwen2.5-coder:14b-q4_K_M

# Modelo rápido — para tareas simples
# VRAM: ~2.5 GB | Velocidad: ~100+ tok/s
ollama pull qwen3:4b-q4_K_M

# Verificar todos los modelos descargados
ollama list

# Test rápido del modelo principal
ollama run qwen3:8b-q4_K_M "Responde solo: tool calling test OK"
8
Configurar Gemini API Key (gratis)
1,500 requests/día — $0 — Fallback para tareas complejas

Obtener la API Key

  1. Ve a aistudio.google.com con tu cuenta Google
  2. Haz clic en "Get API Key" → "Create API Key"
  3. Selecciona "Create API key in new project"
  4. Copia la key (empieza con AIza...)

Configurar como variable de entorno del sistema (permanente)

PowerShell 7 como Administrador
# Guardar la API key como variable de entorno del sistema (permanente)
[System.Environment]::SetEnvironmentVariable(
    "GEMINI_API_KEY",
    "AIza-TU-KEY-AQUI",
    "User"  # "User" = solo para tu usuario | "Machine" = todo el sistema
)

# Verificar que se guardó (cierra y reabre PowerShell primero)
$env:GEMINI_API_KEY
🪟
Alternativa: Variables de entorno por interfaz gráfica

Presiona Win + R → escribe sysdm.cpl → Opciones avanzadas → Variables de entorno → en "Variables de usuario" haz clic en Nueva → Nombre: GEMINI_API_KEY, Valor: tu key.

Hardening de seguridad en Windows
Permisos NTFS, Firewall, y alternativa a NemoClaw

Permisos NTFS del directorio de configuración

PowerShell 7 como Administrador
# Restringir acceso al directorio .openclaw solo a tu usuario
$path = "$env:USERPROFILE\.openclaw"
$acl = Get-Acl $path

# Remover herencia de permisos
$acl.SetAccessRuleProtection($true, $false)

# Dar acceso completo solo a tu usuario
$rule = New-Object System.Security.AccessControl.FileSystemAccessRule(
    $env:USERNAME, "FullControl", "ContainerInherit,ObjectInherit", "None", "Allow"
)
$acl.AddAccessRule($rule)
Set-Acl $path $acl

# Verificar permisos aplicados
Get-Acl $path | Format-List

Regla de Firewall de Windows para el gateway

PowerShell 7 como Administrador
# Bloquear acceso externo al puerto del gateway de OpenClaw
# Solo permite conexiones desde localhost (127.0.0.1)
New-NetFirewallRule -DisplayName "OpenClaw Gateway - Block External" `
    -Direction Inbound `
    -Protocol TCP `
    -LocalPort 5189 `
    -Action Block `
    -RemoteAddress "0.0.0.0/0"

# Permitir solo loopback
New-NetFirewallRule -DisplayName "OpenClaw Gateway - Allow Loopback" `
    -Direction Inbound `
    -Protocol TCP `
    -LocalPort 5189 `
    -Action Allow `
    -RemoteAddress "127.0.0.1"

Troubleshooting específico de Windows
ProblemaSolución Windows
npm: "no se puede cargar porque la ejecución de scripts está deshabilitada"Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
Ollama no encuentra la GPUActualizar drivers NVIDIA desde nvidia.com/drivers y reiniciar
openclaw no se reconoce como comandoCierra y reabre PowerShell 7. Si persiste: $env:PATH += ";$env:APPDATA\npm"
Variables de entorno no se carganCierra y reabre PowerShell completamente después de setear variables con [Environment]::SetEnvironmentVariable
Docker Desktop no iniciaVerificar virtualización habilitada en BIOS. Ejecutar wsl --update en PowerShell como Admin
Puerto 5189 en usonetstat -aon | findstr "5189" para ver qué proceso lo usa, y cambia el port en openclaw.json
Path con espacios da error en workspaceUsar comillas dobles y barras dobles: "C:\\Users\\Mi Usuario\\.openclaw"
